Competitive Analysis Debug Test

This page will test the competitive analysis processing with the fixed field names.

Instructions:

  1. Make sure the development server is running at http://localhost:3002
  2. Go to the application in another tab
  3. Run a competitive analysis query like: "Nike vs Adidas competitive analysis"
  4. Check the browser console for debug messages
  5. Look for these specific messages:

Expected Debug Flow:

  • 🔥 [CachedEndpointRouter] selectEndpoint called with query: "Nike vs Adidas competitive analysis"
  • 🔥 [CachedEndpointRouter] Auto-selected endpoint: /competitive-analysis
  • 🔥 [AnalysisEngine] About to call dataProcessor.processResults with endpoint: "/competitive-analysis"
  • 🔍 [DataProcessor] Selected processor for /competitive-analysis: CompetitiveDataProcessor
  • 🔥 COMPETITIVE DATA PROCESSOR VALIDATE CALLED 🔥
  • 🚨🚨🚨 COMPETITIVE DATA PROCESSOR PROCESS() CALLED! 🚨🚨🚨
  • [CompetitiveDataProcessor] VALUE VERIFICATION for [Area Name]: ...
  • 🔥 [AnalysisEngine] First processed record value: [should be 0-10, e.g., 6.24]

If you see values like 25+ instead of 0-10:

  • The CompetitiveDataProcessor is not being called
  • Raw Nike market share data is being used instead of calculated competitive advantage score
  • Check the console for any errors in the data processing

Success Indicators:

  • ✅ Values are in 0-10 range (e.g., 6.24, 8.91, 3.45)
  • ✅ Map shows circles of different sizes (based on Nike market share)
  • ✅ Map shows different colors (based on competitive advantage score 0-10)
  • ✅ Legend appears at bottom right with color scale
