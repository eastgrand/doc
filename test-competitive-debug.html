Competitive Analysis Debug Test

Open browser console and visit the app, then run a competitive analysis query like:

  • "Show Nike vs Adidas competition"
  • "Analyze competitive landscape"
  • "Nike competitive analysis"

Look for these debug messages in the console:

  1. 🔥 [CachedEndpointRouter] selectEndpoint called with query
  2. 🔥 [CachedEndpointRouter] Using explicitly specified endpoint OR auto-selection
  3. 🔥 [AnalysisEngine] About to call dataProcessor.processResults with endpoint
  4. 🔍 [DataProcessor] Selected processor for [endpoint]
  5. 🔥 COMPETITIVE DATA PROCESSOR VALIDATE CALLED 🔥
  6. 🚨🚨🚨 COMPETITIVE DATA PROCESSOR PROCESS() CALLED! 🚨🚨🚨
  7. 🔥 [AnalysisEngine] First processed record value: [should be 0-10 range]

Expected Results:

  • Endpoint should be exactly "/competitive-analysis"
  • CompetitiveDataProcessor should be called
  • First record value should be in 0-10 range (e.g., 6.24)
  • Visualization should show circles with different sizes and colors

If CompetitiveDataProcessor is NOT called:

  • Check if endpoint auto-selection is working
  • Check if "/competitive-analysis" is being passed correctly
  • Check if DataProcessor.getProcessorForEndpoint() finds the right processor
